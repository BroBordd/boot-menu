Overview

What it does

01

Pre-framework launch

Hooks into early boot via post-fs-data. Starts before Android and exits cleanly when dismissed.

02

Native rendering

Fullscreen OpenGL ES 2.0 surface at INT_MAX layer. 60fps, no framework, no JVM.

03

Hardware input

Raw evdev. Volume keys to navigate, power to confirm. Full touch support via MT Protocol.

04

Boot actions

System, reboot, recovery, download mode, power off — all with confirmation dialogs.

05

Boot status

Animated progress indicator while Android boots in background. Updates when ready.

06

Extras

Drop any Stratum-based binary into extras/ and it appears in the Advanced menu.

03

Reboot

Development

Adding a device

Support is device-specific. Adding a new device requires forking Stratum and providing a device configuration.

# clone with submodules git clone --recurse-submodules https://github.com/BroBordd/boot-menu cd boot-menu # add your device to Stratum # stratum/devices/<model>/StratumConfig.h — input paths, touch ranges, orientation # stratum/devices/<model>/app.cpp — entry point (copy from src/default.cpp) # stratum/devices/<model>/stratum-boot/ — module source # build bash scripts/build.sh <model> # output # stratum/devices/<model>/out/<model>-boot-menu.zip
